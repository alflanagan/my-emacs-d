1.5. Miscellaneous Optimizations

A grab-bag of configuration options from various startup frameworks, collected together by Bozhidar Batsov.

1.5.1. Drop Left-to-Right Text

Turn off the bidirectional algorithm. This optimizes text display and editing by telling emacs to assume text is left-to-right. Obviously, if you frequently work with right-to-left text, don't do this. 

(setq-default
 bidi-display-reordering 'left-to-right
 bidi-paragraph-direction 'left-to-right)
(setq bidi-inhibit-bpa t)

1.5.2. Smooth Input Experience

Tell emacs not to do fontification (syntax highlighting) during input, which can make input slower. Emacs will wait for idle time to do fontification. 

(setq redisplay-skip-fontification-on-input t)

1.5.3. Better Communicate With External Processes

From the blog entry:

The default read-process-output-max is 64KB, which is still quite conservative. Modern LSP servers like rust-analyzer or clangd routinely send multi-megabyte responses. Bumping this reduces the number of read calls Emacs has to make: 

(setq read-process-output-max (* 4 1024 1024)) ; 4MB

If you have several windows visible, Emacs draws a cursor in each of them – even the ones you’re not working in. It also highlights selections in non-focused windows.

This behavior has miniscule performance impact, but some people find the phantom cursors annoying. I'm one of them. 

(setq-default cursor-in-non-selected-windows nil)
(setopt highlight-nonselected-windows nil)

Save a file with a shebang (#!), and Emacs chmod +x​es it for you. One of those things that should arguably be a default.

1.6. Clipboard Tweaks

More optimizations from the blog post by Bozhidar Batsov.

1.6.1. Keep Clipbard Contents

Here’s a scenario: you copy a URL from your browser, switch to Emacs, kill a line with C-k, and then try to yank the URL you copied earlier with C-y. Gone. The kill replaced it on the clipboard.

This setting makes Emacs save the existing clipboard content into the kill ring before overwriting it: 

(setopt save-interprogram-paste-before-kill t)

1.6.2. De-Duplicate Kill Ring

(setopt kill-do-not-save-duplicates t)

1.6.3. Save Kill Ring

(setopt savehist-additional-variables '(search-ring regexp-search-rhig kill-ring))

1.6.4. Sane Syntax in re-builder (Multiple Configs)

Use reasonable syntax (instead of having to escape everything) for re-builder (M-x re-builder), an interactive tool for developing regular expressions. 

(setopt reb-re-syntax 'string)

1.6.5. Fix find-file-at-point

The ffap function is a.k.a. find-file-at-point, and I only use it for files. So it's a real problem when it tries to interpret the filenama as a URL, so I turn it off. 

(setopt ffap-machine-p-known 'reject)

1.6.6. Filter Completion Suggestions

The default filter for M-x completion shows all commands, whether they apply to the current buffer or not. This setting tells it to filter out commands that have been delared to not apply to this buffer.

  • command-completion-default-include-p: filters commands declared to only apply to modes that aren't enabled in the current buffer.
  • transient-command-completion-not-suffix-only-p: filters commands that have been declared to only act as transient suffixes.
(defun my-completion-predicate (symbol buffer)
  "Determine whether SYMBOL should be offered as a completion in BUFFER.
This version filters out commands that don't apply to BUFFER or are transient suffixes."
  (and (command-completion-default-include-p symbol buffer)
       (transient-command-completion-not-suffix-only-p symbol buffer)))

(setopt read-extended-command-predicate #'my-completion-predicate)

1.7. Easy Customization

Set up separate file for use by emacs customization.

I prefer to set customization options in the use-package declaration for each package, but on-the-fly or local-only customizations will go to this file.

(load custom-file 'no-error 'no-message)

1.8. Secrets File

This local file is for global variables with values which should not be checked in with source control, such as passwords and API keys.

Mine defines a function `define-secret-customizations` which calls `setopt` to set customization options that should be secret. 

(require 'my-secrets)
(define-secret-customizations)

1.9. Uniquify

Uniquify is a part of emacs which is not loaded by default, but it can be ~require~d to change the way buffers are named when editing different files with the same name. From the source file:

Emacs's traditional method for making buffer names unique adds <2>, <3>, etc. to the end of (all but one of) the buffers. This file replaces that behavior, for buffers visiting files and Dired buffers, with a uniquification that adds parts of the file name until the buffer names are unique. 

(require 'uniquify)
(setopt uniquify-buffer-name-style 'post-forward-angle-brackets)

1.13.4. Ido and amx

There are (at least) three major emacs packages to enable auto-completion: "ivy", "company", and "ido". Ivy is full-featured and includes "counsel", which modifies emacs commands to use ivy. However, it looks not to be actively maintained, and I've come to prefer "ido". It's builtin to Emacs, which is convenient.

The "amx" package leverages "ido" to add features to the Emacs execute-extended-command function (M-x). 

(use-package amx
    :config (amx-mode 1))

This package replaces stock emacs completion with ido completion wherever it is possible to do so without breaking things (i.e. what you were probably hoping for when you set ido-everywhere to t). 

(use-package ido-completing-read+
  :config
    (ido-ubiquitous-mode 1))

1.13.10. Treemacs

Set up treemacs, the directory tree sidebar. See treemacs repo for all the gory details. 

(use-package
 treemacs
 :defer nil
 :init
 (require 'cua-base)
 (with-eval-after-load 'winum
   (define-key winum-keymap (kbd "M-0") #'treemacs-select-window))
 :config
 (when treemacs-python-executable
   (treemacs-git-mode 'extended)
   (treemacs-git-commit-diff-mode t)
   (treemacs-hide-gitignored-files-mode t))
 ;; treemacs-git-commit-diff fires async callbacks for all workspace projects,
 ;; including ones not currently visible in the buffer.  When the callback
 ;; tries to annotate a path whose project node doesn't exist in the buffer,
 ;; treemacs-project->position receives nil and crashes.  Guard against that.
 (with-eval-after-load 'treemacs-git-commit-diff
   (advice-add
    'treemacs-apply-single-annotation
    :before-while (lambda (path) (treemacs-is-path path :in-workspace))))
 (treemacs-start-on-boot)
 :custom
 (treemacs-project-follow-mode t)
 (treemacs-filewatch-mode t)
 (treemacs-is-never-other-window t)
 ;; follow mode makes it difficult to find file that are not on-screen
 (treemacs-follow-mode nil)
 (treemacs-follow-after-init nil)

 :bind
 ;; M-0 intentionally omitted here; scoped to winum-keymap in :init above
 ;; to avoid shadowing digit-argument globally.
 ;; C-x t 1 → tab-close-other (Emacs 27+); use C-x t o instead.
 ;; C-x t d → dired-other-tab  (Emacs 28+); use C-x t D instead.
 ("C-x t o" . treemacs-delete-other-windows)
 ("C-x t t" . treemacs)
 ("C-x t D" . treemacs-select-directory)
 ("C-x t B" . treemacs-bookmark)
 ("C-x t C-t" . treemacs-find-file)
 ("C-x t M-t" . treemacs-find-tag))

1.13.22. Language Server Protocol

Not long ago, an editor had to have custom handling written for each language it "knew". This led to a lot of duplication of effort, and bad editor implementations.

Now there are 2 major protocols that can be provided by language authors to communicate information about a language to an editor. The Language Server Protocol allows the editor to connect to an external server process, which provides information about the language.

The Tree-Sitter protocol provides a set of shared libraries which the editor dynamically links to, and that run in the editor's process.

Emacs now supports both methods.

This sets up lsp-mode to integrate with the LSP, and then we use a package to integrate the information into the treemacs sidebar also.

LSP Mode Docs 

(use-package
 lsp-mode
 :defer t
 :commands lsp
 ;; these are the LSP servers I have right now. Customize this for your setup!
 :hook ((sh-mode dockerfile-ts-mode sql-mode typescript-ts-mode) . 'lsp-deferred)
 :custom (lsp-enable-snippet nil) (lsp-warn-no-matched-clients nil))

(use-package lsp-treemacs :after (lsp-mode treemacs))

1.13.26. CMake

Support for CMake build files and CMake-based C/C++ projects.

cmake-mode provides syntax highlighting and indentation for CMakeLists.txt and .cmake files. 

(use-package cmake-mode)

cmake-project integrates CMake projects with Emacs's built-in project.el, enabling project-aware compilation and other commands. 

(use-package cmake-project :after cmake-mode)

cmake-ide wires CMake compile commands into IDE features (flycheck, auto-complete, company) by reading the project's compile_commands.json. 

(use-package cmake-ide :after cmake-mode :config (cmake-ide-setup))

1.13.27. Exec-path-from-shell

Gets path (and other variables) from shell and updates Emacs configuration. Without this, on MacOS, load-path gets set from the system default, not the user's environment.

This sets $MANPATH, $PATH and exec-path from your shell, but only when executed in a GUI frame on OS X and Linux.

See also function exec-path-from-shell-setenv to import other shell variables. 

(use-package
 exec-path-from-shell
 :config
 (when (memq window-system '(mac ns x))
   (exec-path-from-shell-initialize)
   (dolist (var '("SSH_AUTH_SOCK" "SSH_AGENT_PID" "LANG" "LC_CTYPE"))
     (add-to-list 'exec-path-from-shell-variables var))))
